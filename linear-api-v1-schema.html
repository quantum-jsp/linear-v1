Account List

Request a lists of all customer accounts. The response will include high level account details.

Authorizations:
custom-apikey-security
query Parameters
search_your_customer_id
string or null

Unique customer identifier used by the partner/vendor (you)

search_start_date
string or null <date-time>

Limits response to deals submitted on or after a given date (can be used in combination with “search_end_date” parameter)

search_end_date
string or null <date-time>

Limits response to deals submitted on or prior to a given date (can be used in combination with “search_start_date” parameter)

search_status
integer <int32>

Limits response to deals in a specific status, where: 100 = “In Processing” 200 = “In Credit Review” 300 = “Pending Additional Information from Customer” 400 = “Partially Approved” 500 = “Approved” 600 = “Offer Option(s) Sent to Customer” 700 = “Offer Accepted – Pending Information for Closing” 800 = “Contracts Sent to Customer” 900 = “Contracts Executed – Ready for Closing” 1000 = “Booked” 1100 = “Offer Declined by Customer” 1200 = “Rejected” 1300 = “Incomplete” 1400 = “Withdrawn”

search_product_type
integer <int32>

Limits response to deals within a certain product class, where: 100 = “Term Loan” 200 = “Extended Payment Plan” 300 = “Line of Credit” 400 = “Purchase Credit Line” 500 = “Other”

search_booked_status
integer <int32>

Limits response to deals in a specific status, where: 100 = “Open” 200 = “Locked” 300 = “Closed”

skip_records
integer <int32>

Records to skip before starting output; used for pagination (can be used in combination with "take records")

take_records
integer <int32> [ 0 .. 200 ]

Maximum number of records to include in output; used for pagination (can be used in combination with "skip_records") Default value: 200

api-version
string or null
